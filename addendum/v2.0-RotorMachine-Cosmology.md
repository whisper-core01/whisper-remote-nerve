#### 7.2 Core Algorithm
function process_fragment(fragment: &[u8], whisper: &mut Whisper) -> Vec<u32> {
// Step 1: Mutation decision (from Loader/MCE state)
for i in 0..10 {
    if should_mutate(whisper.loader.decide(i)) {
        new_alphabet_id = whisper.loader.pick_alphabet()
        whisper.rotor.rouleaux[i].alphabet_id = new_alphabet_id
        whisper.rotor.rouleaux[i].permutation = generate_permutation(
            whisper.rotor.rouleaux[i].alphabet
        )
    }
}

// Step 2: Signal transformation through rouleaux
mut signal = fragment.to_vec()

for rouleaux in whisper.rotor.rouleaux {
    signal = transform_through_rouleaux(signal, rouleaux)
}

// Step 3: Inter-rouleaux permutation (optional)
if should_permute_rouleaux(whisper.loader.state) {
    signal = apply_rouleaux_permutation(signal, whisper.rotor)
}

// Step 4: Update internal state
whisper.mce.digest(signal)
whisper.vault.store(whisper.rotor.state)
whisper.loader.advance_state()
whisper.rotor.cycle += 1

return signal
}
function transform_through_rouleaux(signal: Vec<u32>, rouleaux: Rouleaux) -> Vec<u32> {
mut output = Vec::new()
for symbol in signal {
    // Find symbol in alphabet
    index = rouleaux.alphabet.find(symbol)
    
    // Look up in permutation
    permuted_index = rouleaux.permutation[index]
    
    // Get output symbol
    output_symbol = rouleaux.alphabet[permuted_index]
    
    output.push(output_symbol)
}

return output
}
function should_mutate(loader_decision: u32) -> bool {
// Pseudo-random but deterministic
return (loader_decision % 3) == 0  // ~33% chance per rouleaux
}
function generate_permutation(alphabet: Vec<u32>) -> Vec<u32> {
// Create permutation based on internal state
let mut perm = (0..alphabet.len()).collect()
shuffle(perm, internal_entropy)
return perm
}

### 14. Technical Justifications

#### 14.1 Why 10 Rouleaux?

* 10^10 possible alphabet assignments (manageable combinatorics)
* 10! = 3.6M permutation reorganizations (strong without being excessive)
* Matches naturally with 10 complex alphabets
* Scales well with 4D fractal branching

#### 14.2 Why These Alphabets?

* Complexity range: 24 (Greek) to 3,000 (Hanzi)
* Different family types: logographic, abjad, abugida, alphabetic
* Permutation spaces: 10^24 to 10^3000 per alphabet
* Prevents pattern exploitation (heterogeneous systems harder to analyze)

#### 14.3 Why WASM?

* **Determinism:** Once seed received, all behavior is reproducible on same input
* **Portability:** Runs on x86, ARM, any architecture with WASM runtime
* **Sovereignty:** No GPU dependency, no external calls, completely self-contained
* **Predictability:** 2-5 seconds for 250MB ensures reasonable throughput
* **Auditability:** Pure code, no hidden I/O, no external state

#### 14.4 Why CPU-Only (No GPU)?

* GPU = vendor lock-in and hardware dependency
* CPU operations sufficient for practical throughput
* Enables deployment on resource-constrained devices
* Maintains true sovereignty (not dependent on specific hardware)
* Lookups scale well: 10^9 ops/sec covers 250MB in seconds

#### 14.5 Why Fragment-Based?

* Allows per-fragment mutation
* Enables per-fragment Lemonade reseeding if needed
* Creates natural processing boundaries
* Aligns with network transmission patterns
* Allows streaming processing (not requiring entire file in memory)

### 16. Limitations and Threat Model

#### 16.1 What Whisper Is NOT

* Not a replacement for symmetric/asymmetric cryptography
* Not a substitute for key management systems
* Not immune to compromised Lemonade seed
* Not suitable for ultra-low-latency, real-time systems
* Not a solution to authentication or identity problems

#### 16.2 Threat Model Assumptions

Whisper assumes:
* Terminal security: Device running Lemonade is trusted
* Seed quality: Lemonade's random generation is truly random (not backdoored)
* WASM integrity: WASM execution environment is untampered
* Physical security: Attacker cannot observe seed generation process
* Statefulness: Whisper-Core's internal state is protected from inspection

#### 16.3 What Whisper Protects Against

* **Mass surveillance:** Configuration changes per-sequence make pattern analysis impossible
* **Archive attacks:** Old ciphertexts cannot help predict future ones
* **Cryptanalysis:** Future mathematical breakthroughs don't retroactively compromise prior sequences
* **Centralized dependency:** No reliance on cloud services or external infrastructure
* **Hardware surveillance:** CPU-only design prevents GPU/coprocessor backdoors

#### 16.4 What Whisper Does NOT Protect Against

* **Compromised Lemonade:** If seed generator is backdoored, entire system is compromised
* **Terminal compromise:** Malware on device can observe all operations
* **Physical attacks:** Cold boot attacks, side-channel analysis during execution
* **Quantum computing:** If cryptographically-relevant quantum computers become practical, implications are unclear (as with all post-quantum cryptography)
